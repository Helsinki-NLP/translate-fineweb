| Computer Networks – ARP<br/><br/>Address Resolution Protocol (ARP) is a procedure for mapping a dynamic Internet Protocol address (IP address) to a permanent physical machine address in a local area network (LAN). The physical machine address is also known as a Media Access Control or MAC address.<br/><br/>The job of the ARP is essentially to translate 32-bit addresses to 48-bit addresses and vice-versa. This is necessary because in IP Version 4 (IPv4), the most common level of Internet Protocol (IP) in use today, an IP address is 32-bits long, but MAC addresses are 48-bits long.<br/><br/>ARP works between network layers 2 and 3 of the Open Systems Interconnection model (OSI model). The MAC address exists on layer 2 of the OSI model, the data link layer, while the IP address exists on layer 3, the network layer.<br/><br/>ARP can also be used for IP over other LAN technologies, such as token ring, fiber distributed data interface (FDDI) and IP over ATM.<br/><br/>In IPv6, which uses 128-bit addresses, ARP has been replaced by the Neighbor Discovery protocol.<br/><br/>How ARP works<br/><br/>When a new computer joins a LAN, it is assigned a unique IP address to use for identification and communication. When an incoming packet destined for a host machine on a particular LAN arrives at a gateway, the gateway asks the ARP program to find a MAC address that matches the IP address. A table called the ARP cache maintains a record of each IP address and its corresponding MAC address.<br/><br/>All operating systems in an IPv4 Ethernet network keep an ARP cache. Every time a host requests a MAC address in order to send a packet to another host in the LAN, it checks its ARP cache to see if the IP to MAC address translation already exists. If it does, then a new ARP request is unnecessary. If the translation does not already exist, then the request for network addresses is sent and ARP is performed.<br/><br/>ARP broadcasts a request packet to all the machines on the LAN and asks if any of the machines know they are using that particular IP address. When a machine recognizes the IP address as its own, it sends a reply so ARP can update the cache for future reference and proceed with the communication.<br/><br/>Host machines that don't know their own IP address can use the Reverse ARP (RARP) protocol for discovery.<br/><br/>An ARP cache size is limited and is periodically cleansed of all entries to free up space; in fact, addresses tend to stay in the cache for only a few minutes. Frequent updates allow other devices in the network to see when a physical host changes their requested IP address. In the cleaning process, unused entries are deleted as well as any unsuccessful attempts to communicate with computers that are not currently powered on.<br/><br/>ARP spoofing and ARP cache poisoning<br/><br/>Any LAN that uses ARP must be wary of ARP spoofing, also referred to as ARP poison routing or ARP cache poisoning. ARP spoofing is a device attack in which a hacker broadcasts false ARP messages over a LAN in order to link an attacker's MAC address with the IP address of a legitimate computer or server within the network. Once a link has been established, the target computer can send frames meant for the original destination to the hacker's computer first as well as any data meant for the legitimate IP address.<br/><br/>ARP spoofing can have serious impacts on enterprises. When used in their simplest form, ARP spoofing attacks can steal sensitive information. O enderezo físico da máquina tamén se coñece como enderezo MAC ou Media Access Control.<br/><br/>O traballo do ARP é esencialmente traducir enderezos de 32 bits a enderezos de 48 bits e viceversa. Isto é necesario porque na versión IP 4 (IPv4), o nivel máis común de Protocolo de Internet (IP) en uso hoxe en día, un enderezo IP é de 32 bits de longo, pero os enderezos MAC son de 48 bits de longo.<br/><br/>O ARP funciona entre as capas de rede 2 e 3 do modelo de interconexión de sistemas abertos (modelo OSI). O enderezo MAC existe na capa 2 do modelo OSI, a capa de enlace de datos, mentres que o enderezo IP existe na capa 3, a capa de rede.<br/><br/>O ARP tamén se pode usar para IP sobre outras tecnoloxías LAN, como o token ring, a interface de datos distribuídos por fibra (FDDI) e o IP sobre ATM.<br/><br/>En IPv6, que usa enderezos de 128 bits, ARP foi substituído polo protocolo Neighbor Discovery.<br/><br/>Como funciona o ARP<br/><br/>Cando un novo ordenador se une a unha LAN, atribúeselle un enderezo IP único para usar para a identificación e comunicación. Cando un paquete entrante destinado a unha máquina host nunha determinada LAN chega a unha pasarela, a pasarela pídelle ao programa ARP que atope un enderezo MAC que coincida co enderezo IP. Unha táboa chamada caché ARP mantén un rexistro de cada enderezo IP e o seu enderezo MAC correspondente.<br/><br/>Todos os sistemas operativos dunha rede Ethernet IPv4 manteñen unha caché ARP. Cada vez que un host solicita un enderezo MAC para enviar un paquete a outro host na LAN, comproba a súa caché ARP para ver se a tradución de enderezo IP a MAC xa existe. Se o fai, entón unha nova solicitude de ARP é innecesaria. Se a tradución non existe xa, entón envíase a solicitude de enderezos de rede e realízase ARP.<br/><br/>ARP emite un paquete de solicitude a todas as máquinas da LAN e pregunta se algunha das máquinas sabe que está a usar ese enderezo IP en particular. Cando unha máquina recoñece o enderezo IP como propio, envía unha resposta para que ARP poida actualizar a caché para futuras referencias e proceder coa comunicación.<br/><br/>As máquinas de hospedaxe que non coñecen o seu propio enderezo IP poden usar o protocolo ARP inverso (RARP) para o descubrimento.<br/><br/>Un tamaño de caché ARP é limitado e é periodicamente limpo de todas as entradas para liberar espazo; de feito, os enderezos tenden a permanecer na caché só por uns minutos. As actualizacións frecuentes permiten que outros dispositivos da rede vexan cando un host físico cambia o seu enderezo IP solicitado. No proceso de limpeza, as entradas non utilizadas son eliminadas, así como os intentos fallidos de comunicarse con ordenadores que actualmente non están encendidos.<br/><br/>ARP spoofing e envelenamento de caché ARP<br/><br/>Calquera LAN que use ARP debe ter coidado coa suplantación de ARP, tamén coñecida como enrutamento de veleno ARP ou envelenamento de caché ARP. O ARP spoofing é un ataque de dispositivo no que un hacker difunde mensaxes ARP falsas a través dunha LAN co fin de vincular o enderezo MAC dun atacante co enderezo IP dun ordenador ou servidor lexítimo dentro da rede. Unha vez que se estableceu unha ligazón, o ordenador de destino pode enviar primeiro os fotogramas destinados ao destino orixinal ao ordenador do hacker, así como calquera dato destinado ao enderezo IP lexítimo.<br/><br/>A suplantación de ARP pode ter graves impactos nas empresas. Cando se usan na súa forma máis sinxela, os ataques de suplantación de ARP poden roubar información sensible.
