| Transactions<br/><br/>All writes to Datomic databases are protected by ACID transactions.<br/>Transactions are submitted to the system's Transactor component, which<br/>processes them serially and reflects changes out to all connected<br/>peers. This document describes transactions in detail.<br/><br/>Transaction structure<br/><br/>Datomic represents transaction requests as data structures. This is a<br/>significant difference from SQL database, where requests are submitted<br/>as strings. Using data instead of strings makes it easier to build<br/>requests programmatically.<br/><br/>A transaction is simply a list of lists and/or maps, each of which is<br/>a statement in the transaction.<br/><br/>List forms<br/><br/>Each list a transaction contains represents either the addition<br/>or retraction of a specific fact about an entity, attribute, and value, or the<br/>invocation of a data function, as shown below.<br/><br/>Map forms<br/><br/>Each map a transaction contains is equivalent to a set of one or more<br/>:db/add operations. The map may include a :db/id key identifying<br/>the entity data that the map refers to. It may additionally include<br/>any number of attribute, value pairs.<br/><br/>{:db/id entity-id<br/>attribute value<br/>attribute value<br/>... }<br/><br/>Internally, the map structure gets transformed to the list structure.<br/>Each attribute/value pair becomes a :db/add list. If you did not<br/>provide an entity-id, Datomic will generate a temporary entity-id for<br/>you.<br/><br/>The map structure is supported as a convenience when adding data. As a<br/>further convenience, the attribute keys in the map may be either<br/>keywords or strings.<br/><br/>Identifying entities<br/><br/>There are three ways to specify an entity id:<br/><br/>a temporary id for a new entity being added to the database<br/><br/>an existing id for an entity that's already in the database<br/><br/>an identifier for an entity that's already in the database<br/><br/>They are described in the sections below.<br/><br/>Temporary ids<br/><br/>When you are adding data to a new entity, you identify it using a<br/>temporary id. If you send in two bits, the OR output will be the maximum of those two bits, and the AND output will be the minimum of those two inputs. That's the majority.<br/><br/>The first known example of a monotone function whose circuit reduces from exponential size to polynomial size by using NOT gates was more complicated than a simple majority function.
