| Transactions<br/><br/>All writes to Datomic databases are protected by ACID transactions.<br/>Transactions are submitted to the system's Transactor component, which<br/>processes them serially and reflects changes out to all connected<br/>peers. This document describes transactions in detail.<br/><br/>Transaction structure<br/><br/>Datomic represents transaction requests as data structures. This is a<br/>significant difference from SQL database, where requests are submitted<br/>as strings. Using data instead of strings makes it easier to build<br/>requests programmatically.<br/><br/>A transaction is simply a list of lists and/or maps, each of which is<br/>a statement in the transaction.<br/><br/>List forms<br/><br/>Each list a transaction contains represents either the addition<br/>or retraction of a specific fact about an entity, attribute, and value, or the<br/>invocation of a data function, as shown below.<br/><br/>Map forms<br/><br/>Each map a transaction contains is equivalent to a set of one or more<br/>:db/add operations. The map may include a :db/id key identifying<br/>the entity data that the map refers to. It may additionally include<br/>any number of attribute, value pairs.<br/><br/>{:db/id entity-id<br/>attribute value<br/>attribute value<br/>... }<br/><br/>Internally, the map structure gets transformed to the list structure.<br/>Each attribute/value pair becomes a :db/add list.
